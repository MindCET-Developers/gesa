System Architecture

📂 Request Flow
  1. 1
    Browser → Vercel Edge

    Request arrives at Vercel. Static pages served from CDN; dynamic via RSC.

  2. 2
    Server Component renders

    Page component calls content getters (e.g. getHomeContent()).

  3. 3
    Content Layer

    Getter returns seed data (now) or fetches from Sanity GROQ (after CMS setup).

  4. 4
    HTML streamed to browser

    Fully server-rendered. Client components hydrate for interactivity (Header scroll, VideoDialog, NewsletterForm).

🔄 Content Swap Point

The entire content abstraction is in lib/content/index.ts. Components never import from seed.ts directly — they only call async getters.


Switching to Sanity = swapping getter bodies only. Type signatures and return shapes are identical, guaranteed by TypeScript.

// lib/content/index.ts — before (seed)
export async function getHomeContent() {
  return homeContent; // from seed.ts
}

// after (Sanity)
export async function getHomeContent() {
  return sanityClient.fetch(homeQuery);
}

SEO Migration Strategy

מעבר מ-Wix ל-Next.js — שמירה על דירוגים ו-backlinks קיימים.

⚠️ הסיכון הגדול: Google רואה שינוי פלטפורמה כ"migration event" ומוריד דירוגים זמנית (2–8 שבועות). המפתח הוא שמירת הדומיין, redirects תקינים, ו-sitemap מעודכן.

📅 לוח זמנים לcutover
  1. 1
    לפני הcutover — ייצוא נתוני Wix

    Google Search Console → Performance → Export. שמור את הURLs והדירוגים הנוכחיים כ-baseline.

  2. 2
    לפני הcutover — בדיקת כיסוי URLs

    ייצא את כל ה-URLs מ-GSC ובדוק שכל אחד מהם מפנה או קיים ב-Next.js. אפשר לעשות את זה עם wget --spider או כלי כמו Screaming Frog.

  3. 3
    הוסף דומיין ל-Vercel

    Vercel Dashboard → Project Settings → Domains → הוסף globaledtechawards.org. Vercel מספק CNAME או A record.

  4. 4
    Cutover — עדכן DNS

    בספק הדומיין: עדכן A record/CNAME להצביע ל-Vercel. TTL התפשטות: 24–48 שעות. עשה זאת בלילה ביום חמישי/שישי.

  5. 5
    יום ה-cutover — GSC

    הגש מחדש את ה-sitemap ב-Search Console: https://globaledtechawards.org/sitemap.xml. אמת ownership של הדומיין.

  6. 6
    2–4 שבועות אחרי — ניטור

    Index Coverage ב-GSC. אם יש URLs עם שגיאה 404 — הוסף redirect ל-next.config.ts ופרסם. ירידה קצרה בדירוגים היא נורמלית.
